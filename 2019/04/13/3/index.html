分析误差

我们已经上次的任务是估测神奇宝贝进化后的cp值，在实验中，发现不同的模型中Average error on testing data是不断变化的。那我们最关心的还是如何在最终的testing data中有好的表现，所以我们必须根据每次的误差大小进行调整，这就需要我们先对误差的来源有一个了解。误差来自于两部分：bias和variance。

variance和bias

什么是variance呢，说白了就是散射程度，相当于你去靶场射击，你的枪法很差，子弹孔十分散乱，但是子弹孔的中心还是靶心位置。
那什么事bias呢，就是你是个神枪手，基本上所有的子弹孔都落在一个很小的区域，但是你是个斜眼，所以你最后很遗憾，这个很小的区域偏离靶心。

Diagnosis：

那说白了，在机器学习中怎么判断误差来源于bias还是variance呢？如果你的模型通过训练，还是不能让training examples的error减小，那么说明你bias偏大，Underfitting了如果你现在可以很好地控制training examples的error可是在testing data中表现得很差，那么你现在就是overfitting了。

For bias：

  • 我们可以考虑更多的元素进去，在神奇宝贝中，比如考虑一下身高，体重会不会有影响。
  • 或者我们可以考虑更加复杂的model，比如换成二次项，三次项试试。

For variance

  • 我们可以多加一些例子，更多的例子可以让抖动更小，单个极端的例子不会对function造成太多的影响，但并不是一直有用。
  • 使用正则化，让function变平滑。

模型选择

我们最终要选择的模型肯定是希望bias和variance都很小，但是我们怎么去验证现在的模型就是最佳模型呢？
肯定有人会说：testing data中error小的function就是最优解啊，但是这其实也是错误的，因为testing data其实也不是最终的data，可能你在testing data中表现很好，但是一到最后的real data中，你就是个弟弟！所以这里面又用到了一个方法：交叉验证法！

交叉验证法

做法呢就是我们把Training set分成两个set，一个用做真正的Training set，一个用来选model，叫做Validation set
当然也可以更复杂一点，N-fold Cross Validation 你可以把Training set分成n份，然后在交叉验来验去…
